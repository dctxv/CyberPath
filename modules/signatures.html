SIGN &

A signature proves two things at once: who wrote it, and that not one character changed. Try to forge one — every way — and get caught.

SIGNATURE // BOUND TO THE MESSAGE FINGERPRINT
{{ n.ch }}
SCAN 01 // VERIFY

CHECK IT AGAINST A PUBLIC KEY.

Two claims get checked at once: it really came from Alice, and nobody touched it. Pick a public key, then verify.

VERIFIED // ALICE // UNALTERED — identity and integrity confirmed in one move.
SIGNATURE INVALID // CONTENT ALTERED — the message fingerprint no longer matches the one sealed in the signature. Change the content, break the seal.
IDENTITY MISMATCH // KEY DOES NOT VERIFY — the signature made with Alice's private key won't verify against the imposter's public key.
SCAN 02 // FORGERY

SIGN A FAKE MESSAGE AS ALICE.

You only hold Alice's PUBLIC key, not her private one. Try to sign as her anyway.

PRIVATE KEY REQUIRED // NOT HELD — only the holder of the private key can produce a signature. The public key can only check it. This is RSA run in reverse.

IDENTITY AND INTEGRITY, TOGETHER.

You now have secrecy (encryption), a tamper-detector (hashing), and proof-of-identity (signatures). Real security stacks all three, live, in the first half-second of every secure connection. Want to watch them assemble?
